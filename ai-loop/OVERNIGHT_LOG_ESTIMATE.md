1. **[의심되는 최우선 버그] 인원수 구간 경계에서 총액 비단조성**
   PAX_TIERS(1~9/10~29/30~49/50+)와 enbtMarginTiers가 그룹 전체 단가에 "절벽식"
   할인율을 적용(한계·구간별이 아니라 전원에게 소급 적용)하기 때문에, 구간 경계를
   막 넘는 순간(예: 29명→30명) 오히려 총액이 감소할 수 있음.
   실측 검증(방콕, 이코노미/ICN/평시 가정, 항공+유류+ENBT마진만 비교):
   - 29명: (항공 475,000 + 유류 266,000)×29 + ENBT마진(150,000×29) = 25,839,000원
   - 30명: (항공 450,000 + 유류 252,000)×30 + ENBT마진(138,000×30) = 25,200,000원
   - → 사람이 1명 늘었는데 관련 항목 합계가 639,000원 감소. 다른 항목(관광/보험/현지마진)
     증가분(~25만원)으로는 상쇄 안 되어 전체 총액도 순감소 추정.
   - 9→10명, 49→50명 경계에서도 동일 패턴 발생 가능성 높음.
   → 견적서상 "인원이 늘었는데 총액이 준다"는 것은 담당자/고객 모두에게 신뢰도 저하
     요인이며 실무적으로도 어색함. GPT에게 (a) 이게 실제 버그로 볼 수 있는지
     (b) 고친다면 어떤 방식이 안전한지(예: 계단식→한계구간식 재계산, 혹은 경계에서
     내려가지 않도록 하한 가드) 자문 요청 예정.

2. **[중간] 차량 라인 아이템이 인원수와 무관하게 "차량 1대" 요금 고정**

   차량비가 소형/대형버스 1대분 그대로 청구됨. 실제로는 인원수에 따라 여러 대 필요.
   단, 버스 정원(좌석 수) 기준값이 destinationRates/기존 코드 어디에도 없어 이를
   고치려면 새로운 가정(예: 대형버스 45인승)을 코드에 넣어야 함 — 이는 "임의로
   단가/기준 창작" 소지가 있어 신중 검토 필요.

3. **[낮음, 안전장치 성격] BIZ_ZONES 목록이 destinationRates와 별도 하드코딩**
   현재는 55개 목적지 전부 BIZ_ZONES(short/mid/long) 중 하나에 정확히 매핑되어 있음을

   "매핑 누락 시 콘솔 경고"만 추가하는 안전장치는 순수 방어적 코드라 리스크 낮음.

4. **[보류 후보] 여행자보험 인당 고정 15,000원**

   "콘텐츠성 원가"뿐 아니라 항공권·호텔·보험료 등 목적지 고정원가에도 동일하게
