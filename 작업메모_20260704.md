## 아직 해결되지 않은 것 (이전 세션에서 파악, 미착수)

이건 오늘 손대지 않은, 더 큰 구조적 이슈입니다. 우선순위 순:

1. **⚠️ 견적/문의 데이터가 서버로 전송되지 않음** — 방문자가 견적을 넣거나 문의를 남겨도 `localStorage`에만 저장되어 같은 브라우저가 아니면 회사(관리자)가 절대 확인할 수 없음. 실사용 시 리드(잠재고객) 유실 위험이 가장 큼. 백엔드/DB 연동 필요.
2. **관리자 로그인이 클라이언트 사이드뿐** — 실제 보안 없음 (개발자도구로 우회 가능). 서버 인증 필요.
3. **`ai-loop/itinerary_api.py`가 로컬(localhost:8765)에서만 동작** — 실제 배포 사이트에서 동적 일정/이미지 기능을 쓰려면 별도 서버 호스팅 필요.
4. **Pexels API 키 미설정** (`ai-loop/.env`) — 현재 DuckDuckGo 이미지 검색으로만 폴백 중.
5. **정적 이미지 폴더 일부 빈약** — `이미지/` 하위 폴더 대부분 사진 1장뿐, 일부 목적지는 폴더 자체가 없음.
6. `.claude/settings.local.json`에 다른 PC(`O2DESIGN_10`) 경로와 `mice-estimator-app.vercel.app`, `enbt.co.kr` 허용 기록 — 실제 운영 중인 배포본이나 회사 사이트와의 관계 확인 필요 (동기화 대상인지 참고용인지).

## 참고 사항

- 갤러리 리디자인 사고 이후 교훈: **AI에게 CSS/HTML 파일 전체를 한 번에 재작성시키지 말 것.** 토큰 한도로 잘림 + 클래스명이 몰래 바뀌는 사고 재발 가능. 특정 섹션만 정확히 타겟팅해서 수정하는 방식이 안전함.
- 모든 변경은 Playwright 헤드리스 브라우저로 실제 렌더링까지 확인 후 반영함.
- 관리자 어드민(`admin.html`)은 고객용 사이트와 다른 네이비+화이트 톤을 쓰는데, 이는 의도적으로 유지함 (통일 원하시면 다음에 요청).
